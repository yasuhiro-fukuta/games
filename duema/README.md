# トランプデュエル — duema

トランプで遊ぶデュエマ風カードバトル。あなた vs CPU、どちらも54枚フルデッキ(4色×13+JOKER2枚)。
`index.html` をブラウザで開くだけで遊べる(ビルド不要・依存ゼロの1ファイル構成)。

## ルール

- デッキ54枚、シールド6枚、初手5枚。先攻後攻はランダムで、先攻も1ターン目からドローあり
- **手札は土地としてもクリーチャーとしてもプレイできる**
  - 土地は1ターンに1枚。タップでそのスートのマナ1を出す
  - クリーチャーのコスト = 自スート1マナ + 数字ぶんの無指定マナ(例: ♠3 → 合計4マナ)
  - パワー/タフネス = 数字。出したターンは攻撃できない(召喚酔い)
- **タイミング**: クリーチャーの召喚と JOKER 呪文は、そのターン最初のクリーチャーが攻撃する前のみ(バトル開始後は不可)。土地は攻撃後でも置ける
- **攻撃**: 相手プレイヤー(シールドブレイク)or タップ中の敵クリーチャー。
  防御側はアンタップのクリーチャーでブロックできる(ブロックするとタップ)
- **スート相性**(戦闘時、守り側に +0/+1):
  - ♥ vs ♠ → ♠ が +0/+1
  - ♦ vs ♠ → ♦ が +0/+1
  - ♦ vs ♣ → ♣ が +0/+1
  - ♥ vs ♣ → ♥ が +0/+1
- **A(エース)**: J・Q・K との戦闘時 +13/+13 の修正を受ける(絵札キラー)
- **ダメージ蓄積**: 戦闘ダメージはターン終了時まで蓄積し、累計がタフネス以上になったクリーチャーは破壊される(例: ♥6 に ♥3 で攻撃→3点蓄積、続けて ♥4 で攻撃→計7点≧6で相打ちに持ち込める)。ターン終了時にダメージは全回復
- **シールドトリガー**: ブレイクされたカードは手札へ。そのままコストなしで召喚してもよい(JOKERは手札に加わるだけ)
- **JOKER**(すべて0マナ+追加コスト):
  - 4色どのマナも出せる土地としてセット
  - ♠スペードスピア: 自クリーチャー1体を生贄 → 対象のクリーチャー1体破壊
  - ♥ハートデザイア: 自分のシールドを見て、その1枚を手札1枚と入れ替える
  - ♦ダイヤクラック: 自シールド1枚を生贄 → 相手シールド1枚破壊(どちらもSトリガー発生)
  - ♣クローバーミル: 自土地1枚を生贄 → 相手の対象の土地1枚破壊
- **後攻補正**: 後攻は1回だけフリーマリガン(手札5枚を山札に戻してシャッフルし、5枚引き直し)できる。あなたが後攻のときは開始時に引き直すかどうか選べる
- **勝利条件**: シールドが0の相手に攻撃を通す(ダイレクトアタック)。山札が尽きたら負け


## オンライン対戦(Firebase)

決定論エンジン+アクションログ同期(ロックステップ)方式。同じシードと操作列から両ブラウザが同一盤面を再現するため、Firebase Realtime Database には「操作」だけが流れる(無料枠で十分)。

- タイトル画面 →「部屋を作る」→ 5文字コードを相手に伝える → 相手が「部屋に入る」
- リロードしてもアクションログの再生で盤面が復元される
- `index.html` 冒頭の `FIREBASE_CONFIG` に Firebase プロジェクトの設定を入れると有効になる(未設定の間はCPU対戦のみ)

### Firebase セットアップ

1. https://console.firebase.google.com → プロジェクト作成(Analytics不要)
2. 構築 → Realtime Database → データベース作成(ロケーションは asia-southeast1 など、テストモードで開始)
3. ルールを以下に変更(部屋データのみ読み書き可):
   ```json
   { "rules": { "rooms": { "$code": { ".read": true, ".write": true } } } }
   ```
4. プロジェクト設定 → マイアプリ → ウェブアプリを追加 → 表示される `firebaseConfig` を `index.html` の `FIREBASE_CONFIG` に貼る

## 操作

手札をクリック → 土地に置く / 召喚 / JOKER呪文を選択。自分のクリーチャーをクリック → 攻撃対象(相手エリア or タップ中の敵)をクリック。「ターン終了」で相手番へ。
